# File Manager

*A long-term project focused on building a scalable file manager with clean architecture and a clear evolution path from a console application to a desktop application.*

## About the Project

File Manager is a long-term project focused on developing a file manager with an emphasis on clean architecture, extensibility, and gradual functional evolution.

The project begins as a console application that serves as the foundation for a full-featured desktop file manager.

The main goal of the project is to create more than just another file manager. It aims to become a convenient tool for organizing large amounts of data while addressing common file management challenges found in modern operating systems.

Unlike traditional file managers, this project focuses not only on directory-based organization but also on more flexible data management through features such as:

- custom file groups;
- tag-based organization;
- favorite items;
- advanced sorting and filtering;
- fast search;
- local metadata storage;
- further expansion of file management capabilities.

At the same time, one of the primary objectives is to build a high-quality architecture that allows the application to evolve continuously without requiring major rewrites of the existing codebase.

## Tech Stack

- **Language:** C#
- **Framework:** .NET
- **Current UI:** Console
- **Future UI:** Avalonia

## Project Goals

- develop a file manager focused on efficient organization and retrieval of user data;
- build a scalable and maintainable architecture;
- follow the SOLID principles;
- evolve the project without rewriting the core business logic;
- maintain clear separation of responsibilities between application layers;
- enable reuse of the application's core across multiple user interfaces.

## Roadmap

### Console Application ✅

Develop the first fully functional console version of the application.

### Architecture Refactoring 🔄

Refactor the project architecture and eliminate the currently known design issues.

### Functional Expansion ⏳

Continue expanding the functionality of the console application after the architecture refactoring, taking advantage of the improved design and extensibility.

### Desktop Application ⏳

Develop a graphical desktop interface on top of the existing business logic.

### Metadata Storage ⏳

Introduce local storage for user metadata and application settings.

### Future Development ⏳

Continue expanding the functionality and evolve the project into a full-featured file manager.
